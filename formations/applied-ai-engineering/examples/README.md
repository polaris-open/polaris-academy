# Examples — reference solutions (silent mentors)

> **Status: draft v0.1.** Examples have not been built yet.

This folder will hold the **reference examples** for the *AI Incident & Support
Assistant* — the same project at four quality bars: **bad · basic · good · 10/10**,
plus **solution notes** explaining the trade-offs behind each decision.

For a self-taught learner, these examples act as **silent mentors**: you build your
own version first, then compare against the 10/10 and see the distance — without
anyone telling you.

## Anti-copy rule

Reference solutions are **calibration tools, not shortcuts**:

- **try first** (produce your own version before opening the reference),
- **compare after**,
- **partial before complete**,
- the value is in the **why** (solution notes), not in ready-made code,
- **never start by copying.**

Until these exist, self-study works with the
[starter](../../../starters/ai-incident-support-assistant/README.md) +
self-assessment + the level [rubric](../levels/01-ai-foundations-for-builders/rubric.md).
The canonical document (Sections 5.6 and 17) tracks this as high-priority backlog.

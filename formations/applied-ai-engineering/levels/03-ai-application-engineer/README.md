# Level 3 — AI Application Engineer

> **AI Application Engineer — planned.**
>
> **Status: draft v0.1.** Materials for this level have not been built yet.

This level turns the assistant into an **operable, measured AI application**: it
ingests operational docs, **retrieves evidence (and measures whether it got the
right one)**, answers **with citations and limits**, uses **controlled tool calling
(tool calling is not Agent)**, and exposes a **Minimum Operable API**. It is run as a
mini-program in sprints, not a short module.

The full architecture (sprints, modules, labs, anti-patterns, artifacts, rubric,
blockers) is defined in the canonical document, Section 11:

- [../../canonical/formacao-polaris-applied-ai-engineering-v0.1.md](../../canonical/formacao-polaris-applied-ai-engineering-v0.1.md)

The didactic materials (module pages, lab guides, troubleshooting, FAQs, reference
solutions) will be built later, after Levels 1–2 are validated with real learners.

> Reminder: **RAG is not a layer**, and you do not generate answers over retrieval
> you have not measured. Start at
> [Level 1](../01-ai-foundations-for-builders/README.md).

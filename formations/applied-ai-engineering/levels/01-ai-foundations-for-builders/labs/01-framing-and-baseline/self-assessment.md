# Lab 01 — Self-assessment

Grade yourself **honestly**. There is no grader at Level 1 — self-assessment is the
exercise. Compare your work with the [examples](./examples/excellent-submission.md)
before scoring.

## Scale

```text
0 = not done
1 = weak
2 = acceptable
3 = strong
```

## Criteria

Give each item a score from 0 to 3.

| # | Criterion | Score (0–3) |
|---|---|---|
| 1 | I understood the problem | |
| 2 | I defined a persona / user | |
| 3 | I described the current workflow | |
| 4 | I defined a baseline without AI | |
| 5 | I defined a simple metric | |
| 6 | I identified privacy risks | |
| 7 | I explained why I have not used an LLM yet | |
| 8 | I ran the starter (`make setup/test/check`) | |
| 9 | I recorded reproducible evidence | |
| 10 | I updated `progress.md` | |

**Total: ___ / 30**

## Interpretation

- **0–10 — redo it.** The core (problem, baseline, metric, privacy) is missing.
- **11–20 — acceptable but fragile.** It holds together but has clear gaps; revisit
  the weakest 2–3 criteria.
- **21–26 — good.** Solid Level 1 work; tighten the metric or baseline argument.
- **27–30 — excellent for Level 1.** Clear problem, defensible baseline, honest
  privacy note, reproducible evidence — and you can say why AI is not needed yet.

> "Excellent for Level 1" is a direction of maturity, not a certificate. It does not
> mean production-ready and it does not make you a specialist.

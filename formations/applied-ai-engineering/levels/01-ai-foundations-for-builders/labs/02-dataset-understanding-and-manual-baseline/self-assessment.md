# Lab 02 — Self-assessment

Grade yourself **honestly**. There is no grader — self-assessment is the exercise.
Compare your work with the [examples](./examples/excellent-submission.md) before
scoring.

## Scale

```text
0 = not done
1 = weak
2 = acceptable
3 = strong
```

## Criteria

Give each item a score from 0 to 3.

| # | Criterion | Score (0–3) |
|---|---|---|
| 1 | Can I explain what each dataset field means? | |
| 2 | Can I identify at least three recurring patterns? | |
| 3 | Can I name at least two ambiguous cases? | |
| 4 | Did I define a deterministic baseline (3–5 rules)? | |
| 5 | Did I avoid using LLM/RAG/Agent? | |
| 6 | Did I define one metric? | |
| 7 | Did I write the baseline's limitations? | |
| 8 | Did I avoid real data (synthetic only)? | |
| 9 | Did I update `progress.md`? | |
| 10 | Could another learner reproduce my steps? | |

**Total: ___ / 30**

## Interpretation

- **0–10 — redo it.** The core (data understanding + deterministic baseline) is missing.
- **11–20 — acceptable but fragile.** It holds together but has clear gaps; revisit
  the weakest 2–3 criteria.
- **21–26 — good.** Solid Level 1 work; tighten the rules or the metric argument.
- **27–30 — excellent for Level 1.** Data understood from evidence, a deterministic
  baseline with honest limits, and a clear reason why AI is not needed yet.

> "Excellent for Level 1" is a direction of maturity, not a certificate. It does not
> mean production-ready and it does not make you a specialist.

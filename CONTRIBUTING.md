# Contributing

> **Status: draft v0.1.** Thank you for your interest in Polaris Academy.

This repository is in an early draft. Organization-wide rules (the full
contribution process, templates and policies) live in the shared
[`polaris-open/.github`](https://github.com/polaris-open/.github) repository. This
file only adds what is specific to Polaris Academy.

## How to contribute

- **Start small.** Small, focused contributions are easier to review and more
  likely to land. A typo fix, a clearer sentence, a better troubleshooting entry —
  all welcome.
- **Open an issue or discussion first** for anything larger than a small fix, so we
  can align before you invest time.
- **Be respectful.** See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Non-negotiable rules

- **Synthetic data only.** Every dataset and example must be synthetic.
- **No secrets.** Never commit API keys, tokens, passwords or credentials.
- **No real PII.** No names, emails, phone numbers or document IDs of real people.
- **No customer data.** No real production logs or customer records.
- **No corporate data.** No internal or confidential company data.

If a contribution would add any of the above, it will not be merged.

## The canonical document

The canonical formation document is the architecture, not the course. Do not rewrite
it in place — propose changes through a decision log (see
[docs/decisions/](docs/decisions/README.md)).
